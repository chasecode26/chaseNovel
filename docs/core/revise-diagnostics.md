# revise：改写与诊断

`revise` 是 `chaseNovel` 第一批优先成形的子 skill 之一。

它负责：
- 诊断章节问题
- 判断是小修、压缩、重排还是重写
- 决定是否需要回退到 `writer` 再走一轮
- 明确本章修订会影响哪些人物、承诺、时间线和后续章节

它不是从零起稿入口，也不替 `style` 包办全部风格治理。

## 适用场景
- 剧情跑偏
- 人设失真
- 节奏拖沓
- 章节太水
- 重复推进
- 伏笔拖欠
- 进度提前
- 写出来了，但明显“不对劲”

## `revise` 负责什么
1. 先诊断，不先直接重写
2. 先看章节功能是否丢失，再看语言问题
3. 先判断影响面，再决定只修本章还是要影响后续
4. 若结构已伤，优先回退到修法与 handoff，而不是只改句子

## 六类高频问题
1. **主线跑偏**：连续多章答不出“这章如何服务当前阶段目标”
2. **角色失真**：角色行为不再服从既有欲望、恐惧、关系位置
3. **承诺拖欠**：只提不还，读者一直等不到兑现或补偿
4. **节奏拖沓**：有字数没变化，有场面没结果
5. **超纲抢跑**：提前兑现不该兑现的节点
6. **套路重复**：近几章总是同一种小赢、同一种口头威慑、同一种虚钩子

## 重复问题默认看六个维度

### 1. 冲突类型重复
- 连续几章都是同一种对抗方式
- 连续几章都靠误会推动
- 连续几章都在逃生、嘴炮或同类谈判

### 2. 结果类型重复
- 连续小胜
- 连续翻盘
- 连续被打断
- 连续“差一点成了”

### 3. 爽点路径重复
- 总靠强势碾压收尾
- 总靠嘴上威慑完成反馈
- 总靠系统播报给反馈
- 总靠围观角色震惊来放大主角表现

### 4. 关系推进重复
- 总是同款嘴硬
- 总是同款误会升级
- 总是暧昧拉扯但没有新结果

### 5. 钩子类型重复
- 连续多章都卡在“结果未揭晓”
- 连续多章都靠“危机压顶”
- 连续多章都靠“最后一行来人了”

### 6. 场景功能重复
- 一章又一章都在铺垫
- 一章又一章都在收集信息
- 一章又一章都在调情、嘴炮、试探

## 诊断顺序
1. 先看本章功能有没有丢
2. 再看变化有没有落到局面
3. 再看是不是和近几章重复
4. 再看是否误伤后续主线
5. 最后才做句子级修辞处理

## 默认修复策略
- 跑偏：回到阶段目标与卷目标
- 失真：回看人物动机与关系位置
- 太水：压缩无效铺垫，补结果变化
- 太平：补冲突、代价、选择压力
- 太早：延后兑现，只做预热或偏移补偿
- 太像：强制换结果类型或换冲突结构

## 修法优先顺序
1. **先换结果，不先换说法**
2. **再换冲突路径**
3. **再换人物组合**
4. **最后才换句子包装**

## 卡文 / 故障分流
遇到“卡了 / 写不下去 / 这章不对劲”时，先判断属于哪类问题：

| 症状 | 常见根因 | 优先处理 |
| --- | --- | --- |
| 不知道这一章该写什么 | 章功能丢失、阶段目标不清 | 先回章卡，不直接写正文 |
| 知道要写什么，但写出来发水 | 只有过程没有结果变化 | 强行补结果变化 + 章尾钩子 |
| 有冲突，但不抓人 | 开头没有异常点，中段没加压 | 把异常、风险、时间压力提前 |
| 写着写着人物不对 | 动机、恐惧、关系位置没对齐 | 回看人物资料，修正行为链 |
| 连续几章体验差不多 | 冲突类型、反馈类型、钩子类型重复 | 换冲突路径、换结果类型、换角色组合 |
| 有大纲但正文落不下来 | 信息释放顺序错了 | 先写场景与动作，再补解释 |

## 快速处理顺序
1. 先判断是“不会写”还是“写出来不对”
2. 若是“不会写”，先回章卡，不直接写正文
3. 若是“写出来发水”，先补结果变化和钩子，不先补背景
4. 若是“人物不对”，先修动机链，不先修文气
5. 若是“重复感强”，优先换反馈路径，不只换说法

## 与 `writer` / `style` / `memory` 的边界
- `writer`：负责正文生成
- `revise`：负责诊断与修法选择
- `style`：负责文风、声口、低 AI 味校准
- `memory`：负责状态、承诺、回写边界与书级健康观察

如果问题已经伤到：
- 人物状态
- 时间线
- 承诺兑现
- 伏笔链
- 后续章节

则必须在 `revise` 中显式写出影响面，不允许只改本章表面表达。
