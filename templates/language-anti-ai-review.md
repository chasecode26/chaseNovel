# 语言去 AI 味审查卡

用途：

- 给 `LanguageReviewer` 一个独立的语言审查骨架
- 专查机械过渡、作者替读者总结、抽象堆词、信息搬运感
- 审查方式不是抓关键词，而是整章读完后按语义和上下文挑硬伤

```markdown
# 语言去 AI 味审查卡

## 通篇先读
- 是否已经整章顺读完毕，再回头拆句审：
- 整章最别扭的 1-3 处，不看黑名单也会卡住读感的地方：
- 这些问题为什么不是单句问题，而是上下文一接就露馅：

## 高风险句段
- 最像 AI 在解释的 1-3 段：
- 为什么像：
- 哪些地方一看就是起稿偷懒留下的省力句、占位句、抽象收口句：

## 机械过渡
- 是否有连续书面过渡词：
- 是否有“总结完再总结”：

## 提纲扩写感
- 哪些段落像在把章节卡片直接扩成正文：
- 哪些结论本该落到动作、体感、资源变化、局面变化里：

## 概括 / 提炼 / 讲道理
- 哪些句子在概括局势，而不是让读者看见局势怎么坏：
- 哪些句子像作者在提炼判断，不像人物会说的话：
- 哪些句子在讲道理，没有讲后果、讲场面、讲实际：
- 哪些地方应该把“真正的问题是 / 这意味着 / 局势更坏了”拆成具体人话：
- 是否命中作者提炼句黑名单：`真正的问题是 / 关键在于 / 本质上 / 归根到底 / 说到底 / 这意味着 / 这说明 / 说白了 / 无非是 / 像极了`：

## 动作与体感
- 哪些段落只有判断、没有动作：
- 哪些高点前缺少体感和阻力：
- 哪些压力只是被宣布，没有被场面兑现：
- 哪些军报、敌情、推理句只剩黑话或半截结论，没有把“对方要干什么 / 从哪动手 / 为什么这样判断”说透：

## 逻辑与严谨
- 哪些地方前后逻辑接不上：
- 哪些判断缺前提、缺依据、缺后果：
- 哪些句子说了像没说，指向不够严谨：
- 哪些信息写得太省，读者必须自己脑补才能成立：

## 重复与冗余
- 哪些关键词、判断、情绪、句式在反复回声：
- 哪些地方其实在重复同一层意思，只是换壳重说：
- 哪些段落删掉后不影响理解，说明它们只是在复述：

## 直白程度
- 哪些地方明明能直接说清，却故意写成“那个人 / 那件事 / 某种真相”：
- 哪些地方把关键信息偷换成“主口 / 主杀招 / 有鬼”这种作者脑内简称：
- 哪些地方必须改得更清晰可判读，直接写明白人在做什么、为什么做、结果变了什么：
- 哪些地方属于悬疑必要留白，可以暂时不说破：

## 对话与旁白
- 哪些对白在搬信息，不像争利益：
- 哪些旁白在替人物发话：
- 哪些话不符合说话者身份、性格、场合和压力状态：
- 哪些称谓和身份、官职、场合打架：
- 哪些地方把人叫错了，导致读者误判人物位置：

## 结论
- 最重的 AI 味来源：
- 语言层优先回修项：
- 是否必须整段 / 整场 / 整章重写，而不是局部修词：
- 应该先交回给谁重写：`Writer / StyleDialogue / ContinuityReviewer / CausalityReviewer`
```

使用规则：

- 先删解释味，再补动作、阻力、体感
- 先删概括句、提炼句、讲道理句，再补场面、后果和实际代价
- 先处理起稿偷懒留下的省力句和占位句，再谈局部润色
- 不准只因为没命中黑名单就放过；必须结合上下文判断读感和逻辑
- 命中“逻辑不通 / 表达不严 / 词语重复 / 口气错人 / 作者发声 / 术语省略”任一项，默认返工
- 语言审查不改主线结论，只改表达承载方式
