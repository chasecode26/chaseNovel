# 因果审查卡

用途：

- 给 `CausalityReviewer` 一个最短因果检查骨架
- 专查转折是不是硬推、人物选择是不是成立、结果是不是靠前文推出
- 先整章读完再拆因果，不准只盯某个转折句做局部判断

```markdown
# 因果审查卡

## 通篇先读
- 是否已整章顺读完毕，再回头拆因果：
- 通篇最站不住的 1-3 处因果硬伤：
- 这些问题是缺前提、缺执行过程，还是缺后果承接：

## 转折前提
- 本章关键转折是什么：
- 转折前提在前文有没有布置：
- 哪个前提最薄弱：

## 人物决策
- 主角为什么此时这么选：
- 对手为什么会这样应对：
- 有没有“为了推进只能这么做”的痕迹：

## 结果落地
- 结果是怎么一步步出来的：
- 有没有跳过中间判断或执行过程：
- 有没有作者直接宣布结果：

## 后账影响
- 这个结果会改变什么：
- 下一章最该承接什么：

## 结论
- 最大因果硬伤：
- 必须回修的点：
- 应交回给谁重写：`Writer / CausalityReviewer`
```

使用规则：

- 不问“看起来爽不爽”，先问“为什么会这样发生”
- 若一个结果删掉前文布置也照样能成立，默认因果过薄
- 命中因果硬伤后，要回头检查整章其他判断链有没有同类问题，不能只补一个点
