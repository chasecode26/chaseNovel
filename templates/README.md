# templates

Templates are split by purpose:

- `templates/core/`: project memory templates.
- `templates/launch/`: launch, opening, volume blueprint, and chapter outcome tools.
- `templates/review/`: review, diagnostics, and rewrite handoff templates.

The root keeps only templates with direct value:

- `chapter-card-examples.md`
- `summaries_mid.md`

Old path shims have been removed. Use the layered paths directly.
