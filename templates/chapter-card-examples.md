# 完整章卡示例

用途：
- `/写` 前快速套一个可执行章卡
- 不知道“功能、结果、钩子”怎么落时直接参照

原则：
- 章卡不是摘要，是开写前的执行卡
- 至少写清：功能、冲突、结果变化、角色推进、钩子

## 通用最小模板

```markdown
# 第 X 章写作方案

## 1. 本章章卡
- 本章功能：
- 本章目标：
- 本章冲突：
- 本章结果变化：
- 本章结果类型：
- 本章爽点 / 情绪点：
- 本章主要推进角色：
- 本章关系刷新点：
- 本章承诺推进 / 延后说明：
- 章尾钩子类型：
- 本章章尾钩子：

## 2. 正文草稿
- 开头先落什么：
- 中段必须推进什么：
- 结尾必须留下什么：

## 3. 门禁结论
- 是否有明确推进：
- 是否有结果变化：
- 是否推进角色或关系：
- 是否存在水章风险：
- 是否存在重复风险：

## 4. 需更新的记忆点
- `state.md`：
- `recent.md`：
- `character_arcs.md` / `characters.md`：
- `foreshadowing.md` / `payoff_board.md`：
```

## 示例 1：玄幻 / 修仙反打章

```markdown
- 本章功能：兑现小爽点 + 推进主线
- 本章目标：主角拿回本该属于自己的入山资格
- 本章冲突：执事临时改规，想把名额转给内定弟子
- 本章结果变化：主角保住资格，还让执事当众丢脸
- 本章结果类型：小胜型
- 本章爽点 / 情绪点：第一次正面反打
- 本章主要推进角色：主角
- 本章关系刷新点：围观弟子从轻视转为警惕
- 本章承诺推进 / 延后说明：推进“主角会在宗门翻身”
- 章尾钩子类型：危机压顶型
- 本章章尾钩子：真正负责考核的长老突然点名，让主角单独留下
```

## 示例 2：都市 / 系统流翻盘章

```markdown
- 本章功能：兑现爽点 + 强化系统价值
- 本章目标：主角在会议上拿下原本不可能拿到的客户
- 本章冲突：上司甩锅，同事看笑话，客户极难搞
- 本章结果变化：客户改签给主角，上司失去话语权
- 本章结果类型：资源到手型
- 本章爽点 / 情绪点：系统直接改写现实处境
- 本章主要推进角色：主角
- 本章关系刷新点：同事开始重新估价主角
- 本章承诺推进 / 延后说明：推进“系统会帮主角翻身”
- 章尾钩子类型：结果未揭晓型
- 本章章尾钩子：系统刷新出一个更危险但回报更高的连锁任务
```

## 示例 3：关系推进章

```markdown
- 本章功能：推进关系线
- 本章目标：让两人从互相试探推进到公开偏爱
- 本章冲突：女主被当众针对，男主若出手会暴露立场
- 本章结果变化：男主当众护短，关系第一次半公开
- 本章结果类型：赢局面丢关系型
- 本章爽点 / 情绪点：偏爱落地，第三方见证
- 本章主要推进角色：男女主
- 本章关系刷新点：从暧昧试探进入明确站队
- 本章承诺推进 / 延后说明：推进“偏爱会越来越明显”
- 章尾钩子类型：关系突变型
- 本章章尾钩子：女主回去后发现，男主早替她处理掉更大的麻烦
```

## 使用提醒

- 中盘开始像直线时，先看 `templates/midgame-fatigue-cheatsheet.md`
- 先照结构写，不要先追求文气
