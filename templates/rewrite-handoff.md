# 重写交接卡

用途：

- 给 `Reviewer` 一个判退后必须填写的交接骨架
- 避免只说“这段不行”，却不说明交回给谁、先修什么、改完怎么复核

```markdown
# 重写交接卡

## 1. 判退结论
- 是否阻断：yes / no
- 主故障类型：`逻辑不通 / 表达不严谨 / 词语重复 / 口气错人 / 作者发声 / 术语省略`
- 为什么这不是局部修词能解决的：

## 2. 交回对象
- 交回给谁：`Writer / StyleDialogue / ContinuityReviewer / CausalityReviewer`
- 为什么交回给他：
- 这一轮不该交给谁顺手一起改：

## 3. 重写范围
- 重写单位：`单句 / 单段 / 单场 / 整章`
- 必须保留的事实：
- 必须保留的结果：
- 绝对不能继续出现的写法：

## 4. 重写优先级
- 第一优先先修什么：
- 第二优先再修什么：
- 哪些问题先不要动，避免越修越散：

## 5. 回炉复核点
- 重写回来先看哪 1-3 处：
- 这轮复核最怕漏掉什么：
- 通过标准是什么：
```

使用规则：

- 命中硬伤后默认填写，不准省
- 先定主故障，再定交回对象，不要一口气把所有问题混成一团
- 重写交接必须可执行，不能只写“更通顺一点”“更自然一点”
- 重写回来后，`Reviewer` 必须整章重读，不能只回看被点名的段落
