# 风格护栏

> Primary owner: `style`
> Primary consumer: `writer`
> 写前优先看这份文件。它只保留硬规则、禁区和连续性底线。

## 默认表达

- 默认清晰可读，不故作高深。
- 先写谁在做事，再写为什么做，再写结果变了什么。
- 普通章节不靠谜语句撑气氛。
- 问句出来后，下一句必须答上。
- 一段最多只留一刀提炼，不准段段金句。

## 禁写

- 只给术语，不给动作和后果
- 普通信息绕写成“那个人 / 那件事 / 某种真相”
- “不是……而是……”式作者提炼腔
- “说到底 / 本质上 / 显然 / 毫无疑问”这类作者总结句
- “压迫感 / 拉扯 / 博弈 / 张力 / 信息差”这类分析词直接顶正文
- 上章钉死的时间、地点、人数、伤势、兵器、命令，下一章不交代原因就改口

## 想解释时的替换方向

优先补：
- 谁抬眼了，谁低头了
- 谁不敢接话，谁把手按得更紧
- 哪句命令立刻带来了什么后果
- 谁原本要开口，最后没接上

## 最小连续性先查

写章前先核对四件事：
1. 时间承诺有没有对上
2. 人物、物资、信使、援手的位置和到达时间能不能对上
3. 上章后果现在还在不在
4. `state.md / timeline.md / 章卡` 有没有把钉死的信息写清

## 使用规则

- 先过本文件，再决定是否需要展开 `style.md`、`voice.md`、`character-voice-diff.md`。
- 连续性没对齐时，先修状态和章卡，不要硬写正文。
- 这份文件负责兜底，不负责提供整本书的完整风格画像。
