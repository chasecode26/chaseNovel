# 本章戏剧卡

> 写正文前先填这个，不要直接拿卷纲 / 章卡往正文里怼。  
> 这一页的目标不是“管理任务”，而是把这一章真正值钱的戏想透。

```markdown
# 第 X 章戏剧卡

## 一句话总纲
- 这一章表面发生了什么：
- 这一章真正值钱的冲突是什么：
- 这一章读者最该被哪一下打中：

## 读者体验目标
- 本章最主要的阅读感受（只选 1 个主目标）：紧 / 爽 / 疼 / 憋 / 怕 / 暖 / 酸 / 甜 / 恨 / 燃
- 本章次目标（最多 1 个）：
- 这一章读完后，读者为什么会点下一章：

## 场面骨架
- 开场画面：
- 本章中段最关键的碰撞：
- 本章结尾最该留下的门：

## 人物驱动
- 主角眼下最想立刻解决什么：
- 谁在拦主角：
- 对方怎么拦：
- 主角现在最怕失去什么：
- 对方现在最怕暴露什么：

## 情绪拐点
- 哪一句话 / 哪个动作让局面变了：
- 这一拐之后，谁占上风：
- 本章最该疼的一下落在谁身上：

## 结果变化
- 本章结束后，局面具体变了什么：
- 哪个问题暂时被压住了：
- 哪个更大的问题被抬起来了：

## 代价与后果
- 谁付出了代价：
- 代价是立刻见效，还是埋到下章 / 后文：
- 哪个后果必须写进 `state.md / timeline.md`：

## 绝对不能写成什么样
- 不能把哪场戏写成解释会：
- 不能把哪处冲突写成作者总结：
- 不能让哪个角色说出不符合他身份的话：

## 本章只准留的一刀
- 本章最值钱的一句硬句 / 一组硬句：
```

## 使用规则

- 先填“读者体验目标”，再填“结果变化”。
- 同一章里，读者体验目标最多一个主目标；别又想疼、又想爽、又想燃、又想吓。
- “本章只准留的一刀”是收束提醒，不是让正文里到处写金句。
- 如果这张卡填完仍然只剩“功能推进”，说明戏还没想透，先别起稿。
