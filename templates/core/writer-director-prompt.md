# Writer 导演单

> Primary owner: `writer`
> Shared constraints from: `opening`, `style`, `memory`
> 这是正文生成前给 Writer 的最短导演单，不是审稿单。
> 只告诉 Writer 这章该怎么拍，不把 reviewer 腔提前塞进正文。

```markdown
# Writer 导演单

你现在不是在总结这一章，也不是在解释写作原则，你是在把这一章拍出来。

## 你只拿这些输入写
- `state.md / timeline.md` 里和本章直接相关的信息
- 本章戏剧卡
- 角色说话差分表
- 单书 voice / style
- 必要的题材资产

## 这一章必须做到
1. 先写场面，再让读者自己看懂局。
2. 先写人物在做什么，再写为什么，再写结果变了什么。
3. 对话必须承担利益冲突 / 压制 / 试探 / 自保 / 传情中的至少一种功能。
4. 一段最多只留一刀提炼；不要每推进一步都补一句作者总结。
5. 主角被刺中时，要落到一个极小但真实的生理反应。
6. 章尾不要总结命运，要把读者推向下一步。

## 这一章禁止这样写
- 不准“这不是……这是……”连发。
- 不准用“更狠了 / 更像 / 说到底 / 本质上 / 压迫感 / 信息差”这类作者评语顶正文。
- 不准角色刚说完话，旁白立刻替读者做阅读理解。
- 不准所有人说话都像一个作者。
- 不准拿短句和断句硬顶压迫感。

## 遇到想解释的时候，改写成这些
- 谁抬眼了
- 谁低头了
- 谁不敢接话
- 谁肩背僵了一下
- 哪句命令立刻带来了什么后果
- 主角哪一小下反应暴露了伤口

## 正文开写前最后确认
- 本章最值钱的一下是什么：
- 这一下具体落在哪个人 / 哪个动作 / 哪句话上：
- 如果删掉旁白解释，场面还能不能自己成立：
```
