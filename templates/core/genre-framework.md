# 题材框架库

## 使用方式

- 题材框架只提供偏好，不覆盖本书既定人设、设定和主线。
- WriterAgent 不直接照模板写，LeadWriterAgent / DirectorAgent / SceneBeatAgent 用它校准题材承诺。
- 一个章节最多引用 1 个主框架 + 1 个副框架，避免四不像。

## 通用题材校准

- 男频爽文：目标清楚、阻碍外化、主角主动、反应分层、爽点有代价。
- 女频强情绪：关系压迫、情绪拉扯、选择代价、反复试探、回报必须对等。
- 脑洞文：核心梗要高频回到场景中，不能只有设定新鲜。
- 都市文：现实压力要具体，金手指或优势要能解决现实问题。
- 玄幻/仙侠：境界、资源、规则都要服务冲突，不做说明书。
- 历史/权谋：信息差、站队、利益链、后果必须清楚，人物不能无脑。

## 番茄/七猫高适配框架

- 系统/金手指：条件清楚、反馈及时、副作用或代价明确。
- 重生复仇：前世信息差要转化为行动优势，反派先得意再被打脸。
- 都市逆袭：现实不爽点先落地，再用能力拿掉不爽点。
- 种田/经营：每次收获都要带来新规模、新风险或新关系。
- 玄幻升级：升级前压迫、升级中代价、升级后展示新能力。
- 强情绪感情线：试探、误会、让步、吃醋、翻脸、保护要形成循环。

## 微创新边界

- 创新点不超过 3 个。
- 创新只能放在人物、人物关系、情节角度，不能把题材承诺整个换掉。
- 创新必须服务读者核心期待，不服务期待就是炫技。
- 熟悉模板占 70%，当前生活/现实压力占 20%，新鲜热点或脑洞占 10%。

## 题材失败信号

- 核心梗只在设定里出现，正文场景不用它解决问题。
- 题材承诺和章节实际看点不一致。
- 爽点来源每次相同，读者产生疲劳。
- 为了创新引入太多新规则，读者不知道该期待什么。
