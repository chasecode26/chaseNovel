# 力量体系防通胀约束模板 (Power Economy Constraint)

> 写到50万字以后，力量体系开始出现"通货膨胀"时，强制读取此文件。
> 核心原则：力量体系是一套**有限资源**的经济系统，不是无底洞的数字游戏。

---

## 一、通货膨胀的早期预警信号

出现以下任意一条，立刻亮红灯：

- [ ] 当前卷的"中等反派"，放到第一卷就能当终极BOSS用
- [ ] 主角的战力在三章之内连跳两个大境界
- [ ] 最新登场的路人甲，比第一卷的核心盟友还要强很多
- [ ] 描写力量等级时，开始堆砌新词，读者已经分不清哪个境界更高
- [ ] 主角的某个核心技能已经从"逆天大杀器"变成了"家常便饭"

如果命中2条以上，说明通胀已经发生，必须立刻执行本文件的约束机制。

---

## 二、力量体系的锚定设计法（应在开书时完成）

### 第一步：建立"世界天花板"的具体锚点

开书时，就必须在 `02_knowledge/power_system.md` 中明确写下：

```
世界当前力量天花板：【XXXX境】
天花板的具体标志：能做什么（举例）/ 不能做什么（硬性限制）
主角终局的力量上限：（是否会突破世界天花板？）
突破世界天花板的代价：（如果有，必须是结构性的，不可逆的）
```

**原则**：天花板一旦设定，不得随意上调。如果必须上调（引入更大的宇宙级威胁），必须在卷纲层面提前规划，不能因为写到哪儿了感觉不够用就临时扩容。

### 第二步：建立"阶段锚点"

把整本书的力量成长，划分为若干个**有意义的里程碑**，而非线性的数字刷新：

| 阶段 | 力量锚点 | 对应能力 | 对应最强敌人 | 里程碑事件 |
|---|---|---|---|---|
| 第一卷 | | | | |
| 第二卷 | | | | |
| 第三卷 | | | | |
| ... | | | | 终局 |

**原则**：每个阶段的跨越，必须对应一次结构性的剧情代价（失去某样东西、某人牺牲、完成某个不可逆的选择）。禁止纯数字的"无痛升级"。

### 第三步：为核心技能设定"封印/上限规则"

主角的每一个核心技能，必须在设计时就内置一个"为什么不能无限用"的限制：

- **资源消耗型**：消耗生命力/寿命/精神值，用多了有代价
- **冷却时间型**：需要特定条件才能触发，不能随时调用
- **环境依赖型**：只在特定场合生效，换一个战场就失效
- **渐进损耗型**：每次使用都会造成不可逆的损耗，使用次数有上限

## 三、中后期通胀修复手术

如果开书时没有做锚定，写到中期已经通胀，用以下三把手术刀修复：

### 手术1：引入"维度跃迁"——重新定义规则
不是让主角变弱，而是**引入更大的坐标系**，让原来的通胀数值在新坐标里显得渺小。
- 典型做法："原来你以为的至强境，在宇宙级存在眼里只是萤火虫。"
- 操作要领：这个新坐标系必须有自己的内部逻辑，不能只是换了个名字再度通胀。

### 手术2：核心技能"截肢"——主动付出代价
让主角因为某次重大失败，永久失去一个已经通胀的核心能力，迫使他以更残缺的状态面对下一个阶段的挑战。
- 效果：瞬间重建读者对主角的"脆弱感"，对抗无敌流的审美疲劳。
- 操作要领：这次"截肢"必须是剧情逻辑自然推导出的代价，不能是强行让他消极抵抗或主动送出去。

### 手术3：增加"恒定软限制"
在通胀的战力之上，叠加一个**永远无法用力量解决**的核心困局：
- 比如：主角再强，也强不过一纸天机预言（命运的软性约束）
- 比如：主角的力量越强，对某个他最在乎的人造成的威胁就越大（情感的软性约束）

## 四、力量体系每卷必做的健康检查

每卷结束时，强制回答以下问题：

1. 本卷主角的战力增长幅度，是否超过了"阶段锚点"的预设范围？
2. 本卷新登场的敌人，其最强形态是否在"世界天花板"之内？
3. 核心技能是否已经在本卷内被过度滥用，失去了层次感？
4. 下一卷将面对的最强对手，当前主角能否在"有条件的情况下险胜"？（如果"轻松碾压"则说明已经通胀）
5. 读者在这一卷里，有没有至少一次感受到主角"真的可能输"的恐惧感？
