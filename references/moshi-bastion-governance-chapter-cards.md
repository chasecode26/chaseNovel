# 末世囤货的据点治理章卡群

这份文档只做一件事：

**把末世中后期最容易空掉的“准入、分配、处罚、交易、扩张、闹营”拆成可直接写的一章。**

适用题材：

- 末世囤货
- 极寒末世
- 安全屋 / 据点流
- 废土秩序流

配合文件：

- `references/moshi-bastion-and-order-library.md`
- `references/moshi-character-learning.md`
- `references/moshi-chapter-card-library.md`

## 1. 新人准入章

```markdown
# 第 39 章写作方案

## 1. 本章章节卡
- 本章时间：极寒第 23 日傍晚
- 距上章过去多久：半日
- 本章地点：据点外门禁区
- 本章功能：建立边界 + 首次把人道压力写成规则问题
- 本章目标：让主角第一次公开决定“谁能进据点”
- 本章冲突：外面一家三口濒死求收留，内部成员则担心疾病、内鬼和资源消耗
- 本章结果变化：主角没有全收，也没有全拒，而是立下第一套准入规则和隔离期
- 本章结果类型：翻盘型
- 为什么这次不重复上一章结果：上一章可能是外部威胁，这一章核心是规则生成
- 本章爽点 / 情绪点：主角不是发善心，而是在生死压力下定出能执行的边界
- 本章应触发的既有布局 / 旧伏笔：此前有人质疑主角规则太随意
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、老队友、门禁执行人、求收留者
- 本章关系刷新点：内部第一次感受到“主角说的话会变成制度”
- 本章承诺推进 / 延后说明：推进据点从私产向组织转化
- 章尾钩子类型：关系突变型
- 章尾钩子自动推荐依据：准入决定做完，真正变化会落在内部人心
- 为什么这一钩子适合本章：外人进不进只是第一层，自己人服不服是第二层
- 本章章尾钩子：隔离区刚设好，队里就有人私下给新来者递了药和钥匙

## 2. 正文草稿
- 开头先给门外冻僵和门内沉默的压力
- 中段把争论压在资源、疾病、信任和执行成本上
- 主角必须给出一套可执行的准入机制，而不是一句“先收了再说”
- 结尾把人情与规则的裂缝留在队内
```

## 2. 资源分配章

```markdown
# 第 44 章写作方案

## 1. 本章章节卡
- 本章时间：暴雪连续第 5 日夜
- 距上章过去多久：一日
- 本章地点：据点物资仓和公共餐区
- 本章功能：推进团队变质 + 强化秩序成本
- 本章目标：让主角第一次正面调整物资分配等级
- 本章冲突：燃料不足、药品见底、重伤员与外勤人员都认为自己该优先
- 本章结果变化：主角把“平均分”改成“按风险和岗位分”，稳住眼前但引爆不满
- 本章结果类型：赢局面丢关系型
- 为什么这次不重复上一章结果：上一章可能是生存危机，这一章是内部秩序危机
- 本章爽点 / 情绪点：主角终于不再装公平，而是把生存逻辑讲透并强行落地
- 本章应触发的既有布局 / 旧伏笔：有人偷拿、有人抱怨外勤赚得太多
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、医疗负责人、外勤核心、仓管
- 本章关系刷新点：队伍第一次从“同甘共苦”进入“岗位层级”阶段
- 本章承诺推进 / 延后说明：推进据点从临时避难所进入运行组织
- 章尾钩子类型：选择逼近型
- 章尾钩子自动推荐依据：资源分配规则刚变，接下来肯定会有人试探底线
- 为什么这一钩子适合本章：规则说出来不算数，能不能扛住第一次试探才算
- 本章章尾钩子：夜里仓库警报响起，丢的偏偏是被调低配给那一组人的药

## 2. 正文草稿
- 开头先让“大家都在忍”成立
- 中段让每一方的诉求都带合理性，不搞脸谱化
- 主角必须明确分配标准、执行人、例外条件
- 结尾把规则推向第一次实战考验
```

## 3. 规则处罚章

```markdown
# 第 51 章写作方案

## 1. 本章章节卡
- 本章时间：据点稳定运行后的第 3 周
- 距上章过去多久：半日
- 本章地点：门禁区与公共议事厅
- 本章功能：证明规则高于关系
- 本章目标：让主角第一次按规则处罚自己人
- 本章冲突：核心成员私开侧门接回旧识，差点把陌生人和风险一起带进来
- 本章结果变化：主角没有包庇，而是公开处罚，但也因此让内部情感彻底分层
- 本章结果类型：赢局面丢关系型
- 为什么这次不重复上一章结果：上一章是立规则，这一章是规则第一次砸到自己人头上
- 本章爽点 / 情绪点：主角不再靠人格魅力维持团队，而是真正把制度立住
- 本章应触发的既有布局 / 旧伏笔：之前已出现“规则能否对核心成员生效”的怀疑
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、犯错核心成员、执行队长、围观成员
- 本章关系刷新点：队伍第一次明白“亲近不等于免责”
- 本章承诺推进 / 延后说明：推进主角从领头人转向秩序制定者
- 章尾钩子类型：关系突变型
- 章尾钩子自动推荐依据：处罚本身不是终点，真正的后果在处罚后的人心
- 为什么这一钩子适合本章：有人会因此更服，有人会因此更冷
- 本章章尾钩子：被罚的人没有反抗，却把自己掌握的一条外部补给线主动交了出去

## 2. 正文草稿
- 开头先写违规事实，不要让读者觉得是冤罚
- 中段把“人情”和“规则”冲突顶满
- 主角必须让处罚有条、有据、有成本
- 结尾落在关系裂缝和资源后账
```

## 4. 对外交易章

```markdown
# 第 58 章写作方案

## 1. 本章章节卡
- 本章时间：极寒第 41 日午后
- 距上章过去多久：两日
- 本章地点：废弃收费站外的交易区
- 本章功能：把据点从封闭求生升级成区域博弈
- 本章目标：让主角第一次以组织身份完成对外交易
- 本章冲突：对方愿意给药和电池，但要求知道据点人数、热源和出入频次
- 本章结果变化：主角换回急需资源，同时故意放出一部分假信息
- 本章结果类型：翻盘型
- 为什么这次不重复上一章结果：上一章处理内部，这一章是对外治理
- 本章爽点 / 情绪点：主角第一次不是靠硬抢，而是靠信息差、规则和布置赢下交易
- 本章应触发的既有布局 / 旧伏笔：此前外部势力曾多次试探补给和热源
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、交易对象、记录员、外勤副手
- 本章关系刷新点：外勤副手第一次看见主角在谈判桌上的价值不低于战斗力
- 本章承诺推进 / 延后说明：推进据点向区域性组织升级
- 章尾钩子类型：信息反转型
- 章尾钩子自动推荐依据：交易完成只是表层，真正值钱的是对方信了什么
- 为什么这一钩子适合本章：只有看到对方怎么用这些信息，交易这章才算后劲足
- 本章章尾钩子：对方撤离后，沿路留下的探针明显朝主角故意放出的假点位去了

## 2. 正文草稿
- 开头先写交易前的警戒和人员安排
- 中段让资源谈判、试探问话和假信息投放同时进行
- 主角必须既拿到东西，又不暴露核心命门
- 结尾让“交易成功”直接转成下一轮布局
```

## 5. 外点扩张章

```markdown
# 第 66 章写作方案

## 1. 本章章节卡
- 本章时间：大灾后第 2 个月
- 距上章过去多久：三日
- 本章地点：主据点外的新仓点
- 本章功能：从运行升级到扩张
- 本章目标：让主角第一次决定据点是否扩到第二个点位
- 本章冲突：新点位能提供热源和仓储，但防守线被拉长，内部也担心分兵后主据点变空
- 本章结果变化：主角拿下外点，但被迫建立主点 / 外点双层指挥与轮换制度
- 本章结果类型：保命失机型
- 为什么这次不重复上一章结果：上一章是对外交易，这一章是组织规模升级
- 本章爽点 / 情绪点：主角第一次像经营一股势力，而不是守一间屋子
- 本章应触发的既有布局 / 旧伏笔：仓储压力、对外路线、外勤疲劳
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、外点负责人候选、老成员、巡逻队
- 本章关系刷新点：队伍开始出现“主点派”和“外点派”的潜在分层
- 本章承诺推进 / 延后说明：推进据点从稳定运行进入扩张治理阶段
- 章尾钩子类型：资源争夺型
- 章尾钩子自动推荐依据：新点位到手，必然立刻暴露在争夺与渗透之下
- 为什么这一钩子适合本章：扩张的第一后果从来不是更强，而是更难守
- 本章章尾钩子：新点位的第一夜，巡逻队就在仓顶发现一枚不属于自己人的信号标记

## 2. 正文草稿
- 开头先写扩张的诱惑和主据点现实压力
- 中段把争论落在轮换、防守、调度、通信上
- 主角必须明确谁守主点、谁管外点、谁有最终裁量
- 结尾把扩张的代价立刻落出来
```

## 6. 闹营治理章

```markdown
# 第 74 章写作方案

## 1. 本章章节卡
- 本章时间：据点进入持续低温第 3 个月
- 距上章过去多久：一日
- 本章地点：公共供热点与仓储通道
- 本章功能：检验秩序是否真的能扛住中后期消耗
- 本章目标：让主角处理一次由疲劳、短缺和不信任引发的内部闹营
- 本章冲突：供热优先级调整后，老成员和新成员爆发冲突，仓储通道一度被堵死
- 本章结果变化：主角压住了表面冲突，却不得不承认现有执行层已经不够
- 本章结果类型：保命失机型
- 为什么这次不重复上一章结果：上一章是扩张，这一章是治理能力是否跟得上扩张
- 本章爽点 / 情绪点：主角不是靠镇压，而是靠拆问题、调执行层、补规则把局面拢回来
- 本章应触发的既有布局 / 旧伏笔：分配分层、新人准入、外点扩张后的执行疲劳
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、老成员代表、新成员代表、仓储执行人
- 本章关系刷新点：据点内部正式从“一个团队”变成“一个有层级的组织”
- 本章承诺推进 / 延后说明：推进主角从组织建立者向治理者升级
- 章尾钩子类型：关系突变型
- 章尾钩子自动推荐依据：闹营被压下后，关键不是谁喊得大，而是谁还能继续被信任
- 为什么这一钩子适合本章：治理的真正后果是执行层和人心结构变化
- 本章章尾钩子：事后清点时，主角发现最先煽动冲突的人竟来自自己亲手提拔的执行组

## 2. 正文草稿
- 开头先给闹营现场的热、吵、堵、乱
- 中段把问题拆成供热、分层、执行疲劳、老新矛盾
- 主角必须既止血又改执行结构
- 结尾把问题拉回核心班底
```

## 7. 使用建议

- 若这一章重点是“谁能进据点”，优先套 `新人准入章`
- 若这一章重点是“怎么重新分配稀缺资源”，优先套 `资源分配章`
- 若这一章重点是“规则第一次砸到自己人头上”，优先套 `规则处罚章`
- 若这一章重点是“据点第一次以组织身份对外博弈”，优先套 `对外交易章`
- 若这一章重点是“第二个点位值不值得拿”，优先套 `外点扩张章`
- 若这一章重点是“扩张后内部开始失控”，优先套 `闹营治理章`

一句话原则：

**末世据点治理章最值钱的，不是主角又抢到了什么，而是他怎样让更多人按一套更难的规则继续活下去。**
