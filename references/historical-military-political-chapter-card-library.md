# 历史权谋 / 边境战争的军政冲突章卡群

这份文档只做一件事：

**把“边军、监军、粮官、地方官、朝臣、封赏使者互相掣肘”的场景，直接拆成可写章卡。**

适用题材：

- 历史权谋
- 边境战争
- 军政争霸
- 智勇双全爽文

配合文件：

- `references/historical-command-and-aftermath-library.md`
- `references/historical-character-learning.md`

## 1. 监军夺令章

```markdown
# 第 22 章写作方案

## 1. 本章章节卡
- 本章时间：敌军试探攻城后的当夜
- 距上章过去多久：两时辰
- 本章地点：边城中军帐
- 本章功能：立军令链 + 压缩主角退路
- 本章目标：让主角第一次正面处理“监军要越过主帅夺令”
- 本章冲突：监军拿圣旨要求立刻出营追击，主角判断这是诱敌
- 本章结果变化：主角没有公开抗旨，但把执行层牢牢卡回自己手里
- 本章结果类型：翻盘型
- 为什么这次不重复上一章结果：上一章是打仗，这一章是“谁能决定怎么打”
- 本章爽点 / 情绪点：主角不硬顶圣旨，而是借规则和时机把监军架空
- 本章应触发的既有布局 / 旧伏笔：监军此前抢军报、催战、插手哨骑
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、监军、亲兵校尉、宿将
- 本章关系刷新点：宿将开始承认主角不只是会打，也敢控盘；监军由轻蔑转成记仇
- 本章承诺推进 / 延后说明：推进主角坐稳边军实际指挥链
- 章尾钩子类型：危机压顶型
- 章尾钩子自动推荐依据：刚稳住军令，下一刻必须让判断接受现实验证
- 为什么这一钩子适合本章：监军服不服不重要，关键是敌军是不是真的按主角判断来
- 本章章尾钩子：夜探回报，敌军伏兵果然埋在北坡谷口

## 2. 正文草稿
- 开头先写帐中座次、谁持印、谁先开口
- 中段把冲突落到追击路线、夜间军纪、哨骑回报时差
- 主角不能只靠嘴赢，必须调动亲兵、宿将、军法官形成执行闭环
- 结尾直接落敌军伏兵，证明主角判断值钱

## 3. 门禁结论
- 是否有明确推进：是
- 是否有结果变化：是
- 是否推进角色或关系：是
- 是否符合平台快反馈要求：是
- 是否存在时间线冲突：否
- 是否存在水章风险：低
- 是否存在重复风险：低
- 是否存在中盘疲劳风险：低
```

## 2. 粮官卡脖章

```markdown
# 第 31 章写作方案

## 1. 本章章节卡
- 本章时间：出兵前一日午后
- 距上章过去多久：半日
- 本章地点：转运仓、军营账房
- 本章功能：把后勤写成冲突，而不是背景
- 本章目标：让主角发现有人故意用账目和调拨卡死自己
- 本章冲突：粮官说按册无误，军中却只拿到七成粮；地方官又说马草未到
- 本章结果变化：主角没拿到完整补给，但抓住了一条能反查上游的错账线
- 本章结果类型：保命失机型
- 为什么这次不重复上一章结果：上一章处理的是战场压力，这一章处理的是后方掣肘
- 本章爽点 / 情绪点：主角在一堆账册、封条、调令里看出谁在做局
- 本章应触发的既有布局 / 旧伏笔：此前粮册差额、地方官拖字诀
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、粮官、转运使、军中掌书记
- 本章关系刷新点：军中账房开始真正归心，地方系统对主角敌意坐实
- 本章承诺推进 / 延后说明：延后短期出兵爽点，推进“后方也在打仗”
- 章尾钩子类型：信息反转型
- 章尾钩子自动推荐依据：本章的爽点来自查出错账背后的更高层手
- 为什么这一钩子适合本章：真正的问题不是少了多少粮，而是谁敢在这个时候少
- 本章章尾钩子：最后一张补拨批文上，盖的是朝中刚刚示好的那位大员私印

## 2. 正文草稿
- 开头先写军中对“快发粮”的焦躁
- 中段用过磅、对册、封条、车次把问题具体化
- 主角必须通过细节而不是旁人告密看出问题
- 结尾让问题从地方层级上升到朝堂层级
```

## 3. 地方官逼军章

```markdown
# 第 36 章写作方案

## 1. 本章章节卡
- 本章时间：敌军退后二日
- 距上章过去多久：一日
- 本章地点：边郡官署
- 本章功能：把“军”和“地”拉到同一桌上撕扯
- 本章目标：让主角面对地方官公开拿民生、税粮、城防资源逼军队让步
- 本章冲突：郡守要求立即撤出军粮仓，优先保本地豪族秋税；军中则必须继续备战
- 本章结果变化：主角保住了军仓，却被地方官联名上奏弹劾
- 本章结果类型：赢局面丢关系型
- 为什么这次不重复上一章结果：上一章查后勤，这一章是军地利益正面冲突
- 本章爽点 / 情绪点：主角第一次在地方官场用政治话术压住对方，而不只是靠兵
- 本章应触发的既有布局 / 旧伏笔：地方豪族此前推诿徭役、郡守观望
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、郡守、豪族代表、军中老校尉
- 本章关系刷新点：地方系统从敷衍变成公开对抗；老校尉第一次认主角能处理文场
- 本章承诺推进 / 延后说明：推进军政一体掌控线
- 章尾钩子类型：关系突变型
- 章尾钩子自动推荐依据：这章重点不是仓库留下没，而是地方官接下来如何报复
- 为什么这一钩子适合本章：军中能压地方一次，不代表能压住后面的奏章和舆论
- 本章章尾钩子：当夜就有驿骑南下，带走了三封联名弹章

## 2. 正文草稿
- 开头先写官署里谁坐着、谁站着、谁带刀进门
- 中段让冲突落在税粮、民怨、仓位、军需，不空谈忠君爱民
- 主角得赢得现实，但必须付出政治代价
- 结尾让“地方官输了桌面，赢了后手”显形
```

## 6. 抚恤闹营章

## 1. 本章章节卡

- 本章目标：让主角第一次处理“打赢了，但底下不认这场赢”

- 本章结果类型：保命失机型

- 本章爽点 / 情绪点：主角不是靠砍人压场，而是靠认账、补锅、追责把军心重新拢住

- 本章关系刷新点：底层军心从敬主角勇，转成看主角敢不敢担责
- 本章承诺推进 / 延后说明：推进主角从战将向真正统军者转变
- 章尾钩子类型：关系突变型
- 章尾钩子自动推荐依据：这章重点是“谁还愿不愿意继续跟你”
- 为什么这一钩子适合本章：闹营压住只是第一层，真正麻烦是查出来后该动谁
- 本章章尾钩子：追查抚恤名册时，主角发现错漏最多的一栏正是自己最信任的人经手

## 2. 正文草稿
- 开头先给伤营、哭声、点名错漏的实感
- 中段让军心崩口不是靠喊口号，而是靠具体错账、迟发、死人名字写错
- 主角必须先认责任，再追责任
- 结尾把底层危机引回主角核心圈

## 7. 使用建议

- 若这一章重点是“军权被监军越线”，优先套 `监军夺令章`
- 若这一章重点是“粮草账面正常但实际断供”，优先套 `粮官卡脖章`
- 若这一章重点是“军地利益正面冲突”，优先套 `地方官逼军章`
- 若这一章重点是“胜后怎么分功不撕裂”，优先套 `战后分功撕裂章`
- 若这一章重点是“封赏里藏着拆阵”，优先套 `封赏藏刀章`
- 若这一章重点是“底层军心因抚恤失序崩口”，优先套 `抚恤闹营章`

一句话原则：

**军政冲突章最值钱的，不是谁吼得更大声，而是谁能把资源、命令、人心和后手同时压进一个场面。**
