# `memory` / State and rhythm

`memory` 负责定义书级状态、节奏观察与回写边界。

这份合同只回答三件事：
1. 写章前最少要读什么状态
2. 写完后哪些状态必须回写
3. 节奏判断优先看哪些信号

## 写前最小读取面

写下一章前，至少确认：
- 当前卷 / 阶段位置
- 当前主冲突
- 主要人物位置与关系状态
- 上一章直接后果是否仍在
- 下一章建议方向

最小输出应能回答：
- 现在写到哪一卷、哪一阶段
- 现在最该推进什么，不该误推进什么
- 上一章留下的伤势、命令、关系、资源变化有没有延续

## 节奏观察面

`memory` 关心的不是句子好不好看，而是推进有没有失真。

优先观察：
- 当前卷目标
- 当前阶段位置
- 最近一次兑现 / 反转 / 关系推进
- 当前节奏预警
- 下一阶段建议

常见节奏预警：
- 连续多章只搬信息，不推动结果
- 该兑现的承诺长期拖欠
- 冲突升级不足，章节功能重复
- 人物关系变化已经发生，但状态层没有被记录

## 回写边界

写完一章后，至少应把这些变化回收到状态层：
- 当前卷 / 阶段位置是否变化
- 主冲突是否升级、转向或阶段性解决
- 主要人物位置、关系、立场是否变化
- 下一章准备方向是否变化
- 节奏风险是否新增或解除

如果变化已经影响后续可写性，但没有进入状态层，就视为回写不完整。

## 读取优先级

1. schema memory truth source
2. 必要的 markdown 状态摘要
3. 当前章卡 / planning context
4. 上一章直接后果

若 markdown 与 schema 冲突，以 runtime truth source 为准。

## 对外口径

常用查询面可以压缩为两类：

### `/状态`
输出：
- 当前卷 / 阶段
- 当前主冲突
- 已埋伏笔
- 待回收伏笔
- 主要人物位置
- 下一章建议方向

### `/卷节奏`
输出：
- 当前卷目标
- 当前阶段位置
- 最近一次兑现 / 反转 / 关系推进
- 当前节奏预警
- 下一阶段建议
