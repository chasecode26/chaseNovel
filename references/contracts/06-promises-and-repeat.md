# `memory` / Promises and repeat

这份合同负责两类长期风险：
1. 承诺 / 伏笔有没有拖死
2. 章节推进有没有开始重复

它服务于 `memory`，但会被 `writer`、`revise`、`style` 共同消费。

## 承诺面

这里的承诺包括：
- 明说过的目标、命令、约定
- 已经埋下的伏笔
- 读者已经被诱导期待的兑现点
- 角色已经立下、但尚未完成的关系或行动结果

最小观察输出应区分：
- 当前活跃承诺
- 高风险拖欠项
- 本卷必须兑现项
- 可延期但必须预热项

## 兑现判断

判断一个承诺能不能继续拖，不看“还能不能以后再写”，先看：
- 读者是否已经被连续提醒太久
- 当前卷目标是否还支撑继续拖延
- 如果现在不兑现，是否必须补新的预热或代价
- 提前兑现会不会压缩后续更大的转折

最小输出应能回答：
- 指定承诺的兑现方案
- 最适合兑现的章节窗口
- 提前兑现 / 延后兑现的风险
- 可搭配的钩子与结果反馈

## 防重复面

重复不是题材重复，而是推进手法和结果感重复。

优先检查：
- 当前章是否又在重复同一种冲突结构
- 近章是否反复用同一类解释、同一类嘴炮、同一类压迫或反转
- 看似推进了，实际是否只是换说法重演旧结果
- 当前是否已经存在升级变体

最小输出应区分：
- 当前章与近章的重复风险对照
- 重复维度判断
- 是否已有升级变体
- 替代路径建议

## 回写要求

以下情况必须回写到承诺 / 重复观察层：
- 新承诺被明确立下
- 旧承诺被兑现、转向或作废
- 同类冲突已经连续复用，开始形成重复风险
- 某类推进方式已经失效，需要切换手法

## 对外口径

### `/承诺`
输出：
- 当前活跃承诺
- 高风险拖欠项
- 本卷必须兑现项
- 可延期但需预热项

### `/兑现`
输出：
- 指定承诺的兑现方案
- 最适合兑现的章节窗口
- 提前兑现 / 延后兑现的风险
- 可搭配的钩子与结果反馈

### `/防重复`
输出：
- 当前章与近章的重复风险对照
- 重复维度判断
- 是否已有升级变体
- 替代路径建议

### `/角色弧`
输出：
- 目标角色当前弧线位置
- 最近变化
- 当前欲望 / 恐惧 / 误解
- 下一次推进建议
- 失真或停滞风险
