## 1. 边关军议章

```markdown
# 第 18 章写作方案

## 1. 本章章节卡
- 本章时间：边关告急后当夜
- 距上章过去多久：半日
- 本章地点：镇北军中军大帐
- 本章功能：显局 + 提压 + 立主角军中位置
- 本章目标：让主角在主战、死守、撤兵三派里拿下决断权
- 本章冲突：监军拿圣旨压人，老将拿军规压人，粮草只够七日
- 本章结果变化：主角没有直接接帅印，但实际军议按他的方案运转
- 本章结果类型：赢局面丢关系型
- 为什么这次不重复上一章结果：上一章是外敌压境，这一章不是再写战报，而是写谁有资格决定怎么打
- 本章爽点 / 情绪点：主角第一次压住老资格将领与监军
- 本章应触发的既有布局 / 旧伏笔：前文埋过的粮道问题、某老将对主角的轻视
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、监军、宿将
- 本章关系刷新点：宿将从看不起转为表面服从，监军从轻慢转为记恨
- 本章承诺推进 / 延后说明：推进“主角将真正接管边军”的核心承诺
- 章尾钩子类型：危机压顶型
- 章尾钩子自动推荐依据：本章刚确定打法，下一章必须让打法立刻遭遇现实考验
- 为什么这一钩子适合本章：军议赢了不等于战场赢了，章尾立刻把纸面方案压进现实
- 本章章尾钩子：探马回报，敌军先锋比预判提前一夜抵达

## 2. 正文草稿
- 开头先落人：谁先坐、谁后坐、谁带甲入帐、谁没卸刀
- 中段把争论压在粮草、城防、军令时限上，不空谈忠义
- 让主角靠一条更高明但更冒险的方案拿下话语权
- 结尾别停在“众人服了”，而是立刻落敌军提前到来的现实压力

## 3. 门禁结论
- 是否有明确推进：是
- 是否有结果变化：是
- 是否推进角色或关系：是
- 是否符合平台快反馈要求：是
- 是否存在时间线冲突：否
- 是否存在水章风险：低
- 是否存在重复风险：低
- 是否存在中盘疲劳风险：低

## 4. 需更新的记忆点
- `state.md`：军议已定，实际指挥权开始向主角倾斜
- `timeline.md`：敌军提前抵达
- `character_arcs.md`：宿将与主角关系从轻视进入压服期
- `payoff_board.md`：边军掌控承诺推进一格
```

## 2. 朝堂试探章

```markdown
# 第 27 章写作方案

## 1. 本章章节卡
- 本章时间：边关小胜后的次日早朝
- 距上章过去多久：一夜
- 本章地点：京中朝堂
- 本章功能：推进权谋线 + 提高站队成本
- 本章目标：让主角在受赏与受制之间做出选择
- 本章冲突：皇帝当众封赏，台谏与门阀同时借机发难
- 本章结果变化：主角拿到了名分，却丢掉了继续装中立的空间
- 本章结果类型：赢局面丢关系型
- 为什么这次不重复上一章结果：上一章是军事结果，这一章是政治后账，不重复打仗反馈
- 本章爽点 / 情绪点：主角在朝堂上第一次不靠武力而靠话术与局势借力自保
- 本章应触发的既有布局 / 旧伏笔：皇帝此前的观察、某门阀对主角的拉拢
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、皇帝、某台谏、某门阀代表
- 本章关系刷新点：皇帝从试用转为借力，门阀从拉拢转为警惕
- 本章承诺推进 / 延后说明：推进“主角终将卷入更高层权力结构”
- 章尾钩子类型：关系突变型
- 章尾钩子自动推荐依据：本章核心不是战果，而是公开站队后的关系后果
- 为什么这一钩子适合本章：朝堂一过，真正的变化会发生在散朝后的私下动作
- 本章章尾钩子：刚出宫门，原本站在中间的老臣派人递来密信，要单独见主角

## 2. 正文草稿
- 开头先写朝堂座次、目光、谁先开口，不急着放大段议论
- 中段让封赏和掣肘同一时间落下，形成甜里藏刀
- 主角必须做选择，不许全拿
- 结尾落在“表面得势，实则站队价码暴涨”

## 3. 门禁结论
- 是否有明确推进：是
- 是否有结果变化：是
- 是否推进角色或关系：是
- 是否符合平台快反馈要求：是
- 是否存在时间线冲突：否
- 是否存在水章风险：低
- 是否存在重复风险：中低
- 是否存在中盘疲劳风险：低

## 4. 需更新的记忆点
- `state.md`：主角已无法继续维持中立姿态
- `character_arcs.md`：皇帝与主角关系进入“借力 + 防范”阶段
- `payoff_board.md`：上位承诺推进，但代价同步抬高
```

## 3. 粮道崩口章

```markdown
# 第 33 章写作方案

## 1. 本章章节卡
- 本章时间：守城第三日傍晚
- 距上章过去多久：两时辰
- 本章地点：边城粮仓、城头、后营
- 本章功能：制造硬危机 + 打断顺推
- 本章目标：让主角意识到真正的问题不是攻城，而是后方断供
- 本章冲突：前线还能守，后方粮道却被自己人截断
- 本章结果变化：城没丢，但守城计划彻底改写
- 本章结果类型：保命失机型
- 为什么这次不重复上一章结果：上一章是正面守住，这一章是守住后发现根本问题转移
- 本章爽点 / 情绪点：不是赢，而是主角在最短时间内看穿“谁在让他输”
- 本章应触发的既有布局 / 旧伏笔：前文提过的粮册差额、地方官拖延
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、粮官、亲兵头领、地方转运使
- 本章关系刷新点：亲兵对主角更死心塌地，地方体系对主角敌意公开化
- 本章承诺推进 / 延后说明：延后短期战功兑现，推进更大层级的敌人浮出水面
- 章尾钩子类型：信息反转型
- 章尾钩子自动推荐依据：本章核心快感在于发现“问题不在城外而在城内”
- 为什么这一钩子适合本章：把外战直接转内斗，能强推下一章
- 本章章尾钩子：查出来最后一批粮车的调令上，盖的是主角自己阵营的印

## 2. 正文草稿
- 开头先给“还能守住”的错觉
- 中段让主角在细节里发现不对，不靠天降情报
- 把危机落到粮、马、夜巡、军心，不写空悬念
- 结尾用调令反转，把矛头刺回自己阵营

## 3. 门禁结论

- 是否有结果变化：是
- 是否推进角色或关系：是
- 是否符合平台快反馈要求：是
- 是否存在时间线冲突：否
- 是否存在水章风险：低
- 是否存在重复风险：低
- 是否存在中盘疲劳风险：低

## 4. 需更新的记忆点
- `state.md`：守城逻辑切换为查内鬼 + 稳军心
- `timeline.md`：粮道断供发生时点
- `foreshadowing.md`：己方印信问题转为关键暗线
- `character_arcs.md`：主角从外战决策者进一步转为军政一体处理者
```

## 4. 盟友背刺章

```markdown
# 第 41 章写作方案

## 1. 本章章节卡
- 本章时间：大战前夜
- 距上章过去多久：一日
- 本章地点：盟军营地
- 本章功能：刷新阵营图 + 提高决战难度
- 本章目标：让主角看清盟友不是来帮他赢，而是来防他赢
- 本章冲突：盟友口头出兵，实则拖延，只想让主角与敌军两败俱伤
- 本章结果变化：主角没有撕破脸，但彻底改写了对盟友的用法
- 本章结果类型：翻盘型
- 为什么这次不重复上一章结果：上一章是对外布局，这一章是对盟友认知翻盘
- 本章爽点 / 情绪点：主角不翻桌，反而借背刺做局
- 本章应触发的既有布局 / 旧伏笔：盟友此前要粮、要名分、要军功的异常动作
- 若未触发，失效或延后原因：无
- 本章主要推进角色：主角、盟友统帅、幕僚
- 本章关系刷新点：盟友从合作方变成可利用但不可托底的对象
- 本章承诺推进 / 延后说明：推进“大局之中无纯盟友”的权谋承诺
- 章尾钩子类型：选择逼近型
- 章尾钩子自动推荐依据：主角识破背刺后，必须立刻决定是先吃敌军还是先卡盟友
- 为什么这一钩子适合本章：本章结尾的关键不是知道真相，而是下一步怎么动
- 本章章尾钩子：主角收到两封急报，一封敌军夜袭，一封盟友暗撤

## 2. 正文草稿
- 开头先给盟友情面，不直接撕破
- 中段靠粮、兵、时机、话术看出背刺逻辑
- 让主角把“被坑”变成“反借”
- 结尾同时压来两难选择

- 是否有结果变化：是
- 是否推进角色或关系：是
- 是否符合平台快反馈要求：是
- 是否存在时间线冲突：否
- 是否存在水章风险：低
- 是否存在重复风险：中低
- 是否存在中盘疲劳风险：低

## 4. 需更新的记忆点
- `state.md`：盟友可信度降为低，临战结构改写
- `character_arcs.md`：主角与盟友关系从合作进入利用阶段
- `payoff_board.md`：联盟承诺破损，转为个人大局承诺
```

## 6. 使用建议

- 若这一章重点是“谁能决定”，优先套 `边关军议章`
- 若这一章重点是“谁在借谁”，优先套 `朝堂试探章`
- 若这一章重点是“外战转内斗”，优先套 `粮道崩口章`
- 若这一章重点是“盟友不是盟友”，优先套 `盟友背刺章`
- 若这一章重点是“赢了也要罚”，优先套 `部将违令章`

一句话原则：

**历史权谋和边境战争的章卡，最值钱的不是打赢，而是每赢一次，局面、关系、代价都跟着变。**
