## 防漂移机制

### 主线防跑偏
- 每章都要能回答：它如何服务当前阶段目标？
- 若连续多章不能回答，说明已经偏航

### 角色防失真
- 关键角色做重大选择前，回看其动机、恐惧、欲望、关系位置
- 角色可以变化，但变化必须有原因与过程
- 若某个核心角色连续多章没有新变化，优先检查是否需要推进 `character_arcs.md`

### 承诺防拖欠
- 读者已经被承诺过的爽点、情绪点、真相点，不能长期只提不还
- 若本章不兑现核心承诺，也应提供预热、偏移补偿或更高一级的新压力
- 卷末前必须清理当卷核心承诺，避免跨卷烂尾感

### 节奏防拖沓
- 同一类型场景不能长时间重复
- 铺垫必须伴随新变化
- 大冲突前的小冲突要逐步升压，而不是原地循环
- 若连续多章没有小兑现或局面变化，优先回看 `arc_progress.md` 与 `payoff_board.md`

### 超纲防跳跃
- 大纲里明确的节点不可提前硬跳
- 可以提前埋线，不能提前兑现
- 可以预热，不能抢跑

## 防重复入口

重复检查已拆到独立文件，避免本页继续膨胀：

- 维度判断、重复窗口、变体标准：详见 `references/anti-repeat-rules.md`
- 自动化扫描：`scripts/anti_repeat_scan.py`

默认原则不变：

- 不只看桥段名，要看读者体验是否同质
- 不只看台词和场景，要看冲突结构、结果类型、反馈路径是否重复
- 若没有升级、没有变体、没有新结果，就按重复风险处理

## 修订与修复任务

“修章”不是笼统重写，要先诊断故障类型。

### 卡文 / 故障诊断分流

当用户说“卡了”“写不下去”“这章不对劲”时，先不要盲写，先判断属于哪一类问题：

| 症状 | 常见根因 | 优先处理 | 对应参考 |
|---|---|---|---|
| 不知道这一章该写什么 | 章功能丢失、阶段目标不清 | 先回 `plan.md`、`state.md`，重写章卡 | `写前流程`、`先定章节功能` |
| 知道要写什么，但写出来发水 | 只有过程没有结果变化 | 强行补“结果变化 + 章尾钩子” | `追读约束`、`references/writing-patterns.md` |
| 有冲突，但不抓人 | 开头没有异常点，中段没加压 | 把异常、风险、时间压力提前 | `番茄风格默认约束`、`references/writing-patterns.md` |
| 写着写着人物不对 | 动机、恐惧、关系位置没对齐 | 回看人物资料，修正行为链 | `角色轮转与出场频率规则` |
| 连续几章体验差不多 | 冲突类型、反馈类型、钩子类型重复 | 换冲突路径、换结果类型、换角色组合 | `references/anti-repeat-rules.md` |
| 有大纲但正文落不下来 | 信息释放顺序错了，先讲设定后讲事件 | 先写场景与动作，再补解释 | `信息释放纪律`、`references/writing-patterns.md` |
| 文字有气势，但总像作者在替场面发话 | 旁白判词过密，动作/称呼/站位承载不足 | 删一半判语，改写成军令措辞、人物反应、礼数失衡、物件细节 | `references/writing-patterns.md`、`assets/genres/09-lishi-quanmou.md` |

### 快速处理顺序

1. 先判断是“不会写”还是“写出来不对”
2. 若是“不会写”，先回章卡，不直接写正文
3. 若是“写出来发水”，先补结果变化和钩子，不先补背景
4. 若是“人物不对”，先修动机链，不先修文气
5. 若是“重复感强”，优先换反馈路径，不只换说法

### 复核判退后的最短动作

如果 `Reviewer` 已经明确命中硬伤，不要停在“再润一下”“再顺一顺”。默认按下面处理：

1. 先写清主故障属于哪一类
   - `逻辑不通`
   - `表达不严谨`
   - `词语重复`
   - `口气错人`
   - `作者发声`
   - `术语省略`
2. 再决定交回给谁
   - 结构和因果问题：优先交回 `Writer / CausalityReviewer`
   - 声口和人物口气问题：优先交回 `Writer / StyleDialogue`
   - 连续性和设定问题：优先交回 `Writer / ContinuityReviewer`
   - 多类问题同时命中：默认回到 `Writer` 主重写，再走 reviewer 复核
3. 再写明白这轮只先修哪一层，不准一边修逻辑一边顺手把书味改散
4. 重写回来后，`Reviewer` 必须整章重读，不能只盯被点名的句子

统一交接骨架见：

- `templates/review/rewrite-handoff.md`

### 阶段复盘最短写法

当问题已经不是单章，而是最近几章一起发水、重复或拖欠时，先别继续堆正文，直接套：

- `templates/review/retro.md`

只写 4 件事：

- 当前疲劳点
- 当前重复点
- 当前拖欠承诺
- 下一阶段纠偏动作

### 常见修复类型

| 问题 | 典型症状 | 优先修法 |
|---|---|---|
| 太水 | 写了很多但局面没变 | 压缩无效段落，强化目标/冲突/结果 |
| 爽点不够 | 读完没反馈、没情绪波峰 | 提前兑现、加强结果、强化反差 |
| 人设崩 | 角色像换了个人 | 回到人物动机，补过渡或重写行为 |
| 节奏拖 | 一直铺垫不爆发 | 抬高压力，提前释放小结果 |
| 信息太多 | 像在讲设定书 | 删减解释，改为冲突中展示 |
| 氛围太重 | 满篇在烘托，但场面没自己站住 | 删连续判断句，保留一刀重话，其余改成动作、反应与硬信息 |
| 钩子太弱 | 章尾没有继续看动力 | 改成未揭晓结果/风险逼近/关系突变 |
| 重复 | 桥段似曾相识 | 更换冲突形态、结果类型或角色组合 |

### 改写流程

1. 先标出必须保留的内容
   - 关键事实
   - 关键伏笔
   - 必须延续到后文的设定
2. 再判断主要故障只有一个还是多个
3. 优先修最影响追读的故障
4. 小修时尽量不扩大影响面
5. 大修后必须重新检查：
   - 人物
   - 时间线
   - 伏笔状态
   - 章尾钩子
   - 后续章节是否被连带影响

## 用户明确说“听不懂 / 不合理 / 没答上”时

这种情况不要先修文气，先修信息和因果。

优先顺序：

1. 先删术语黑话
   - 如 `主口`、`主杀招`、`有鬼`、`压不住`、`接刃`
2. 再把首次术语翻成人话
   - 让读者在同句或下一句知道它是什么、现在干什么、影响谁
3. 再把判断改成 `事实 -> 判断 -> 后果`
4. 再检查问答是否真的答上、军令是否真的能执行
5. 最后才修文气、节奏和章尾

如果做完这 5 步，读者仍要自己补完整意思，默认不通过，不准硬交。

战争文复检最短口径：

- 敌人现在在打哪
- 主角为什么这样判断
- 命令下去后谁先动
- 如果不这么做，先乱的是哪一段
