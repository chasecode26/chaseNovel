## 防漂移机制

### 主线防跑偏
- 每章都要能回答：它如何服务当前阶段目标？
- 若连续多章不能回答，说明已经偏航

### 角色防失真
- 关键角色做重大选择前，回看其动机、恐惧、欲望、关系位置
- 角色可以变化，但变化必须有原因与过程
- 若某个核心角色连续多章没有新变化，优先检查是否需要推进 `character_arcs.md`

### 承诺防拖欠
- 读者已经被承诺过的爽点、情绪点、真相点，不能长期只提不还
- 若本章不兑现核心承诺，也应提供预热、偏移补偿或更高一级的新压力
- 卷末前必须清理当卷核心承诺，避免跨卷烂尾感

### 节奏防拖沓
- 同一类型场景不能长时间重复
- 铺垫必须伴随新变化
- 大冲突前的小冲突要逐步升压，而不是原地循环
- 若连续多章没有小兑现或局面变化，优先回看 `arc_progress.md` 与 `payoff_board.md`

### 超纲防跳跃
- 大纲里明确的节点不可提前硬跳
- 可以提前埋线，不能提前兑现
- 可以预热，不能抢跑

## 结构化防重复检查

重复检查不只看“有没有像”，而要看“读者这次得到的体验是否和前几章高度同质”。

### `repeat_report.md` 建议固定输出

- 本章核心桥段标签
- 本章冲突类型
- 本章结果类型
- 本章爽点/情绪点类型
- 本章章尾钩子类型
- 本章主要人物组合
- 近 3-5 章对应标签对照
- 重复项
- 是否存在升级或变体
- 是否允许重复
- 替代建议

### 必查维度

- **冲突类型重复**：又是同一种对抗、误会、逃生、打脸、谈判
- **结果类型重复**：又是同一种小胜、翻盘、被打断、反杀
- **爽点路径重复**：又靠同一种强势碾压、口头震慑、系统播报、围观惊叹完成反馈
- **关系推进重复**：又是同一种暧昧拉扯、同款嘴硬、同款误会升级
- **钩子类型重复**：连续多章都用同一种“结果未揭晓”或“危险逼近”
- **场景功能重复**：章节都在做铺垫，但没有新增局面

若只是表面桥段相似，但冲突结构、人物位置、结果反馈已经升级，可判为“可接受变体”；否则视为重复风险。

## 修订与修复任务

“修章”不是笼统重写，要先诊断故障类型。

### 卡文 / 故障诊断分流

当用户说“卡了”“写不下去”“这章不对劲”时，先不要盲写，先判断属于哪一类问题：

| 症状 | 常见根因 | 优先处理 | 对应参考 |
|---|---|---|---|
| 不知道这一章该写什么 | 章功能丢失、阶段目标不清 | 先回 `plan.md`、`state.md`，重写章卡 | `写前流程`、`先定章节功能` |
| 知道要写什么，但写出来发水 | 只有过程没有结果变化 | 强行补“结果变化 + 章尾钩子” | `追读约束`、`writing-techniques.md` |
| 有冲突，但不抓人 | 开头没有异常点，中段没加压 | 把异常、风险、时间压力提前 | `番茄风格默认约束`、`writing-examples.md` |
| 写着写着人物不对 | 动机、恐惧、关系位置没对齐 | 回看人物资料，修正行为链 | `角色轮转与出场频率规则` |
| 连续几章体验差不多 | 冲突类型、反馈类型、钩子类型重复 | 换冲突路径、换结果类型、换角色组合 | `结构化防重复检查` |
| 有大纲但正文落不下来 | 信息释放顺序错了，先讲设定后讲事件 | 先写场景与动作，再补解释 | `信息释放纪律`、`writing-techniques.md` |
| 文字有气势，但总像作者在替场面发话 | 旁白判词过密，动作/称呼/站位承载不足 | 删一半判语，改写成军令措辞、人物反应、礼数失衡、物件细节 | `writing-techniques.md`、`templates/genres/09-lishi-quanmou.md` |

### 快速处理顺序

1. 先判断是“不会写”还是“写出来不对”
2. 若是“不会写”，先回章卡，不直接写正文
3. 若是“写出来发水”，先补结果变化和钩子，不先补背景
4. 若是“人物不对”，先修动机链，不先修文气
5. 若是“重复感强”，优先换反馈路径，不只换说法

### 常见修复类型

| 问题 | 典型症状 | 优先修法 |
|---|---|---|
| 太水 | 写了很多但局面没变 | 压缩无效段落，强化目标/冲突/结果 |
| 爽点不够 | 读完没反馈、没情绪波峰 | 提前兑现、加强结果、强化反差 |
| 人设崩 | 角色像换了个人 | 回到人物动机，补过渡或重写行为 |
| 节奏拖 | 一直铺垫不爆发 | 抬高压力，提前释放小结果 |
| 信息太多 | 像在讲设定书 | 删减解释，改为冲突中展示 |
| 氛围太重 | 满篇在烘托，但场面没自己站住 | 删连续判断句，保留一刀重话，其余改成动作、反应与硬信息 |
| 钩子太弱 | 章尾没有继续看动力 | 改成未揭晓结果/风险逼近/关系突变 |
| 重复 | 桥段似曾相识 | 更换冲突形态、结果类型或角色组合 |

### 改写流程

1. 先标出必须保留的内容
   - 关键事实
   - 关键伏笔
   - 必须延续到后文的设定
2. 再判断主要故障只有一个还是多个
3. 优先修最影响追读的故障
4. 小修时尽量不扩大影响面
5. 大修后必须重新检查：
   - 人物
   - 时间线
   - 伏笔状态
   - 章尾钩子
   - 后续章节是否被连带影响
