## 开书前必须先补齐的骨架

都市系统流开书时，至少先锁清：

- 系统怎么改变现实，而不是只会播报
- 主角混哪条现实赛道，规则和门槛是什么
- 奖励怎么落到钱、名、人脉、地位或规则优势
- 信息怎么扩散，谁能压消息，谁会带节奏
- 至少三层反派：眼前对手、制度 / 资本 / 关系型对手、更高层对手

不补这些，后面很容易只剩：

- 系统面板念稿
- 反派轮流瞧不起人
- 爽点全靠围观震惊

## 一、 核心逻辑：规则改变现实

都市系统流的本质是**新规则闯进现实，并立刻改写现实处境**。读者追读的动力来自“即时反馈”和“现实回声”。

重点不是“系统说了什么”，而是：

- 主角因此获得了什么
- 失去了什么顾虑
- 撬动了什么现实局面
- 别人因此必须重新站位

## 二、 写法重点

1. **现实困境要快，系统反馈要落地**
   - 开篇 800 字内建立主角的现实痛点（穷、被欺、失业、被甩）
   - 系统介入后，第一章必须有一次可感知的反馈

2. **每章走完一轮反馈闭环**
   - 困境 → 任务/机会 → 执行 → 即时反馈 → 外界反应 → 新目标
   - 奖励要能立刻影响现实局面，不能只存在系统面板里

3. **反制靠结果和站位变化，不靠围观喊爽**
   - 最有效的反转，是让对手原本依赖的秩序、资源或身份失灵
   - 外界反应要分人写：谁改口，谁转身站队，谁开始害怕，谁还在嘴硬

4. **系统规则简单清晰**
   - 一章只讲一条系统规则，不要一次倒灌所有功能
   - 用事件结果展示规则，不用系统播报解释规则

5. **语言要利落，但不要被压成统一口水腔**
   - 主角可以直接，但不必永远只会一句一顶
   - 配角要按阶层、职业、利害、熟悉度区分口吻
   - 允许潜台词，但读者必须看得出谁在套话、谁在拖延、谁在装没听懂

## 三、 常见失误

| 失误 | 表现 | 修正方向 |
|------|------|---------|
| 系统说明过多 | 开篇列任务栏、商城、属性面板、抽奖机制 | 用一次具体任务展示系统，不要产品说明书 |
| 播报替代剧情 | 全靠【叮！】撑爽点，没有现实反馈 | 每次播报后必须接现实变化 |
| 配角只会震惊 | 围观群众集体复读"恐怖如斯" | 给每个配角具体作用（站队变化、资源变化、信息差） |
| 重复装穷打脸 | 每章都是"被嘲笑→亮身份→震惊" | 换反馈路径：赚钱、资源、信息、人脉、规则反制 |
| 全员统一短视频腔 | 主角、老板、家人、对手说话都一个味 | 按身份、熟悉度、压力和场合拉开声口 |

## 四、 推进技巧

- 一章至少走完一轮：困境 → 任务/机会 → 执行 → 反馈
- 奖励要能立刻影响现实局面
- 打脸不要只靠嘴，要靠结果和站位变化
- 系统升级要带来新能力边界，不只是数值+1
- 每 `10-15` 章最好给一次新玩法或新规则压力，别只重复旧反馈

## 五、 差例 / 好例

**差例**：
> 系统说明了很多功能，还列出了任务栏、商城、属性面板和抽奖机制，秦川看了很久，终于大概明白了。

**好例**：
> 【叮！临时任务：五分钟内拿下这单。】
> 秦川抬头，正好看见一直压着他的经理把合同甩在桌上："你不是很能吗？你去。"

**为什么**：差例像产品说明书。好例把系统直接绑进现实冲突里。都市系统最怕虚，最好一上来就让规则改变现实局面。

## 六、 章尾钩子推荐

| 类型 | 示例 |
|------|------|
| 奖励揭晓 | 结算界面弹出来，秦川的手开始发抖 |
| 打脸反转 | 他刚要说话，门口突然传来一声"秦总" |
| 关键人物到场 | 电梯门开了，走出来的那个人让他瞳孔骤缩 |
| 系统异常 | 【警告：检测到未知干扰源】 |

## 七、 节奏与句式

- **短中句快节奏**，重要判断句和结果句允许稍拉长一拍
- **对话占比 40%-60%**，对白必须有信息量或冲突
- **反转时限**：压抑不超过半章（约 1500 字内必须反转）

## 八、 禁忌表达

- 大段铺垫商业逻辑或设定
- 长时间空转纠结，拖到行动和反馈都不起
- 圣母心、无底线宽恕
- "围观群众都惊呆了"（用具体反应代替概括）
